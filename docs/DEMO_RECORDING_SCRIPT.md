# Magellan Demo 录屏剧本（投资人 / 合作伙伴详细版）

## 0. 这版剧本的目标
这是一份面向 PoC 演示的“可直接照读 + 可直接操作”脚本，核心目标不是展示某个单点功能，而是证明：
1. 系统具备从议题输入到报告资产沉淀的完整闭环。
2. 系统支持人机协同决策（不是纯自动黑盒）。
3. 系统对长耗时、多轮、外部依赖波动有可解释的稳定性设计。
4. 系统已经具备可演示、可扩展到业务化部署的结构基础。

建议总时长：12-15 分钟。

---

## 1. 演示前准备（T-15 分钟）

### 1.1 环境检查
1. 仅保留一个浏览器窗口（关闭通知、插件弹窗、下载提示）。
2. 前后端服务已启动，登录可用。
3. 网络稳定，避免切换 Wi-Fi。
4. 清除敏感信息：真实 API Key、隐私数据、个人路径。

### 1.2 演示数据准备
1. 预置一个长议题（用于展示“标题提炼 + 原始问题保留”）。
2. 准备一个短议题（异常兜底）。
3. 报告列表中至少有 1 份历史成功报告（兜底素材）。

### 1.3 演示账号状态
1. 使用演示账号登录，角色权限正常。
2. 默认语言中文。
3. 保证“头脑风暴”“报告”“自动交易”“设置”菜单都可点击。

## 2. 本场必须证明的“核心点”（建议先背）

### 核心点 A：完整闭环
从“输入议题 → 多智能体协同讨论 → 人工打断干预 → 生成纪要/报告 → 报告导出”。

### 核心点 B：可控的 AI，而非黑盒
支持 HITL（打断并补充、暂停后无补充继续、节点分叉重跑），关键决策可人工介入。

### 核心点 C：知识检索可配置，不是默认全库乱检索
支持“是否启用知识检索 + 选择知识范围（下拉）”，适合合规和成本控制。

### 核心点 D：长议题处理友好
报告列表展示“提炼后的短标题”，详情保留“原始议题”，兼顾可读性和可追溯性。

### 核心点 E：长时任务稳定性
头脑风暴支持长耗时，超时窗口放宽；WebSocket 断开可重连；失败可回退到历史资产演示。

### 核心点 F：国际化与演示一致性
中英切换后关键页面文案一致，不影响核心流程演示。

## 6. 异常场景兜底剧本（必须准备）

### 场景 A：WebSocket 断开
- 话术：
  “当前是连接波动，系统会自动重连。我们先看已沉淀的报告资产，不影响核心演示目标。”
- 操作：
  1. 切到报告列表。
  2. 打开最近成功报告。
  3. 继续讲“资产沉淀和导出能力”。

### 场景 B：模型 4xx/5xx 或超时
- 话术：
  “这属于外部模型瞬时波动。我们的流程设计允许回看历史资产，核心链路仍可演示。”
- 操作：
  1. 停止当前等待。
  2. 切换短议题重开，或直接打开历史报告。
  3. 继续演示导出与复盘能力。

### 场景 C：外部搜索能力不稳定
- 话术：
  “在线检索是增强项，主链路是会议编排 + 人工干预 + 报告沉淀。”
- 操作：
  1. 关闭检索或缩小知识范围。
  2. 继续完成讨论与报告展示。

## 8. 投资人可能会问的 6 个问题（建议回答）

1. **问：你们和普通聊天机器人差异是什么？**  
答：我们重点是“流程化协作 + 人工干预 + 资产沉淀”，而不是单轮问答。

2. **问：如果模型不稳定怎么办？**  
答：流程支持重试、降级和历史资产兜底；演示和业务沟通不会被单次调用阻断。

3. **问：是否支持合规控制？**  
答：知识检索可显式开关与范围控制，便于在不同数据边界下运行。

4. **问：可扩展性如何？**  
答：当前单机 PoC 已稳定，架构可拆分为网关、编排、报告、检索等服务做分布式扩展。

5. **问：商业化优先路径是什么？**  
答：先做“高价值场景的决策辅助和报告自动化”，再逐步开放自动执行能力。

6. **问：为什么先做 PoC？**  
答：先验证用户价值和流程闭环，再投入生产级可靠性，效率更高、风险更可控。
