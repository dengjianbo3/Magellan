## 💡 最佳实践

### 1. 文件命名
- 永久文档：`README.md`, `DESIGN.md`
- 版本文档：`RELEASE_v1.0.0.md`
- 临时文件：`temp_test.sh`, `debug_*.py`

### 2. 目录组织
```
项目根目录/
├── 核心文档和配置 (少而精)
├── 核心脚本 (常用的)
├── tests/ (测试代码)
├── archive/ (历史文档)
└── temp/ (临时文件，.gitignore)
```

### 3. Git规范
- 使用 `git mv` 移动文件（保留历史）
- 使用 `git rm` 删除文件（记录删除）
- 详细的commit message
- 分步提交（归档、删除、优化）
